# Skills

Skills provide procedural knowledge, tool instructions, and workflow guidance for agents. During runtime initialization, DojoAgents loads `*/SKILL.md` files from several directories and exposes them through `skills_list`, `skill_view`, and the skill management tools.

## Default Load Paths

By default, DojoAgents loads:

| Source | Path | Purpose |
| --- | --- | --- |
| User skills | `~/.dojo/skills` | Main personal skill directory |
| Generated skills | `~/.dojo/skills/generated` | Skills generated by memory or tools |
| Built-in skills | `dojoagents/skills/built_in/` | Skills shipped with the package |
| External dirs | `skills.external_dirs` | Extra directories declared in config |
| Plugin dirs | Provided by plugin registry | Plugins may contribute skill directories |

Each skill is a directory containing `SKILL.md`:

```text
~/.dojo/skills/
└── my-skill/
    └── SKILL.md
```

`SKILL.md` should usually include YAML frontmatter:

```markdown
---
name: my-skill
description: When to use this skill.
category: finance
platforms:
  - macos
---

# My Skill

Steps, constraints, and examples.
```

## Configuration

Configure skills in `~/.dojo/agents.yaml`:

```yaml
skills:
  dir: ~/.dojo/skills
  generated_skill_dir: ~/.dojo/skills/generated
  external_dirs:
    - ~/work/shared-dojo-skills
  disabled:
    - old-skill
  platform_disabled:
    dashboard:
      - terminal-heavy-skill
  read_claude_skills: false
```

Field summary:

- `dir`: main user skill directory and the default target for skill management tools.
- `generated_skill_dir`: directory for generated skills.
- `external_dirs`: additional skill roots.
- `disabled`: globally disabled skill names.
- `platform_disabled`: platform-specific disabled skill names.
- `read_claude_skills`: whether to also read Claude Code skills.

## Enable Claude Code Skills Compatibility

Claude Code skills usually live in:

```text
~/.claude/skills
```

Enable compatibility with:

```yaml
skills:
  read_claude_skills: true
```

When enabled, the runtime appends `~/.claude/skills` to the skill search path. The directory is still read by the DojoAgents loader rules: each skill is a subdirectory containing `SKILL.md`.

Example:

```text
~/.claude/skills/
└── spreadsheet-helper/
    └── SKILL.md
```

If you only want to load a specific Claude skills directory, you can skip the global switch and use `external_dirs` instead:

```yaml
skills:
  external_dirs:
    - ~/.claude/skills
```

Difference:

| Method | Use case |
| --- | --- |
| `read_claude_skills: true` | Direct compatibility with the default Claude Code skills directory |
| `external_dirs: [~/.claude/skills]` | Explicit directory control or non-default locations |

## Loading Behavior

- With `agent.lazy_skills: true`, the agent first sees a skill catalog and must call `skill_view` to read full instructions.
- With `agent.enable_skill_cache: true`, parsed skill content is cached in `.skills_cache.json` under the main skill directory.
- `platforms` in frontmatter limits a skill to matching operating systems.
- `requires_tools` in frontmatter requires named tools to be loaded before the skill is available.
- Duplicate skill names load only the first discovered version; runtime path order decides precedence.

## Modules

| Module | Purpose |
| --- | --- |
| `dojoagents/skills/loader.py` | Build the skill prompt block |
| `dojoagents/skills/cache.py` | Cache skill content |
| `dojoagents/skills/manager.py` | Parse, filter, and manage skills |
| `dojoagents/tools/skill_manage.py` | Expose skill create/list/view tools |
| `dojoagents/agent/runtime.py` | Assemble skill search paths |
